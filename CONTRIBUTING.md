# Contributing

Thanks for your interest in contributing!

- Bug reports: open an issue with steps to reproduce and expected vs actual behavior.
- Feature requests: open an issue describing the use case and proposal.
- Pull requests: fork the repo, create a branch, make your changes, and open a PR.

Style
- Keep changes small and focused.
- Follow existing code style and naming.

Testing
- If you add features, include tests or instructions to reproduce manually.

CI
- The repo uses GitHub Actions; ensure any new dependencies are added to `requirements.txt`.

License
- By contributing you agree to the project's license (see `LICENSE`).
